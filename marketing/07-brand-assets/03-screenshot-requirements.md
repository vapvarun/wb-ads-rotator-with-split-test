# WB Ad Manager - Screenshot & Visual Asset Requirements

**Purpose:** Standardize visual assets for marketing
**Use:** Website, WordPress.org listing, social media, documentation

---

## General Guidelines

### Image Quality
- **Resolution:** Minimum 1920x1080 for screenshots
- **Format:** PNG for screenshots, JPG for photos
- **File size:** Optimize for web (under 500KB when possible)
- **Retina:** Provide 2x versions for key assets

### WordPress Admin Screenshots
- Use a fresh WordPress install
- Twenty Twenty-Three or neutral theme
- Default admin color scheme (light)
- Browser zoom at 100%
- No bookmarks bar visible
- No browser extensions visible
- Use Chrome for consistency

### Plugin State
- Demo data should look realistic
- Use placeholder ad images (not competitor brands)
- Ensure all sample text is professional
- No personal information visible
- No lorem ipsum - use realistic content

## File Naming Convention

### Screenshots
`wbam-[area]-[description]-[version].png`
Examples:
- wbam-dashboard-ad-list-v1.png
- wbam-frontend-popup-dark-v2.png
- wbam-pro-classifieds-grid-v1.png

### Social Images
`wbam-[platform]-[purpose]-[variant].png`
Examples:
- wbam-twitter-header-v1.png
- wbam-instagram-carousel-01.png
- wbam-linkedin-feature-analytics.png
